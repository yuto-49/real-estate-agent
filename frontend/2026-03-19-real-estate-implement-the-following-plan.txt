│ Plan to implement                                                            │
│                                                                              │
│ Plan: UX Redesign — 3-Page Architecture + MiroFish-Inspired Simulation       │
│                                                                              │
│ Context                                                                      │
│                                                                              │
│ The platform currently has 6 navigation pages (Search, Users, Negotiate,     │
│ Simulation, Intelligence, System) which fragments the core user journey. The │
│  user wants a simpler 3-page architecture focused on:                        │
│                                                                              │
│ 1. User sets up their investor profile and needs                             │
│ 2. System generates a MiroFish intelligence report with strategy             │
│ recommendations                                                              │
│ 3. User runs multi-scenario negotiation simulations with detailed agent      │
│ personas                                                                     │
│ 4. User compares results across scenarios and picks the best strategy        │
│                                                                              │
│ Additionally, the simulation engine should be enhanced inspired by           │
│ MiroFish's architecture: richer agent personas, multiple scenario variants   │
│ run in parallel, more rounds, price path tracking, and a comparison          │
│ dashboard.                                                                   │
│                                                                              │
│ ---                                                                          │
│ New Navigation Structure                                                     │
│                                                                              │
│ [Dashboard]  [Analysis]  [Simulation]          [⚙ gear icon]                 │
│      /        /analysis     /simulation          (SystemDrawer)              │
│                                                                              │
│ - Dashboard (/) — User profile selection + quick-start actions + recent      │
│ activity                                                                     │
│ - Analysis (/analysis/:reportId?) — Report generation + report viewer +      │
│ embedded search drawer                                                       │
│ - Simulation (/simulation) — Persona builder → scenario selector → multi-run │
│  → results comparison                                                        │
│ - Negotiate (/negotiate) — Kept as hidden route (no nav link), accessible    │
│ from simulation results                                                      │
│ - System — Gear icon in header opens a slide-out drawer                      │
│                                                                              │
│ ---                                                                          │

│ 1.1 Create services/scenario_variants.py                                     │
│                                                                              │
│ Define 6 real-estate scenario variants (inspired by MiroFish's               │
│ ScenarioSpecGenerator):                                                      │
│                                                                              │
│ ┌───────────────────────┬───────────────┬─────────────────────────────────┐  │
│ │       Scenario        │  Description  │         Key Constraints         │  │
│ ├───────────────────────┼───────────────┼─────────────────────────────────┤  │
│ │                       │ Low demand,   │ seller_urgency=high,            │  │
│ │ market_favors_buyer   │ property on   │ competing_offers=0,             │  │
│ │                       │ market 90+    │ initial_offer_discount=12%      │  │
│ │                       │ days          │                                 │  │
│ ├───────────────────────┼───────────────┼─────────────────────────────────┤  │
│ │                       │ Hot market,   │ buyer_urgency=high,             │  │
│ │ market_favors_seller  │ multiple      │ competing_offers=3,             │  │
│ │                       │ offers        │ max_discount=3%                 │  │
│ ├───────────────────────┼───────────────┼─────────────────────────────────┤  │
│ │ balanced_market       │ Normal        │ competing_offers=1,             │  │
│ │                       │ conditions    │ max_discount=8%                 │  │
│ ├───────────────────────┼───────────────┼─────────────────────────────────┤  │
│ │                       │ Buyer pushes  │ initial_offer_discount=15%,     │  │
│ │ aggressive_buyer      │ hard, low     │ buyer_style=aggressive          │  │
│ │                       │ initial offer │                                 │  │
│ ├───────────────────────┼───────────────┼─────────────────────────────────┤  │
│ │                       │ Small         │                                 │  │
│ │ conservative_approach │ increments,   │ max_counter_increment=1.5%,     │  │
│ │                       │ patient       │ min_rounds_before_accept=5      │  │
│ │                       │ negotiation   │                                 │  │
│ ├───────────────────────┼───────────────┼─────────────────────────────────┤  │
│ │                       │ Both sides    │ soft_turn_limit=5,              │  │
│ │ time_pressure         │ want quick    │ deadline_urgency=high           │  │
│ │                       │ close         │                                 │  │
│ └───────────────────────┴───────────────┴─────────────────────────────────┘  │
│                                                                              │
│ Each variant is a dataclass: name, description, constraints: dict,           │
│ max_rounds: int.                                                             │
│                                                                              │
│ 1.2 Create services/persona_generator.py                                     │
│                                                                              │
│ Generate rich agent personas using Claude API. Input: user profile data.     │
│ Output:                                                                      │
│                                                                              │
│ @dataclass                                                                   │
│ class AgentPersona:                                                          │
│     role: str                    # "buyer" or "seller"                       │
│     name: str                    # Generated name                            │
│     personality_type: str        # MBTI-style (from MiroFish)                │
│     negotiation_style: str       #                                           │
│ aggressive/analytical/collaborative/avoidant                                 │
│     risk_tolerance: str          # high/medium/low                           │
│     experience_level: str        # first_time/experienced/professional       │
│     motivations: list[str]       # e.g. ["investment returns", "family       │
│ home"]                                                                       │
│     background: str              # 2-3 sentence bio                          │
│     pressure_points: list[str]   # what makes them concede                   │
│     strengths: list[str]         # negotiation advantages                    │
│                                                                              │
│ For the buyer: derive from user profile (budget, risk_tolerance, life_stage, │
│  goals).                                                                     │
│ For the seller: generate a complementary persona based on property context + │
│  market conditions.                                                          │
│                                                                              │

│         # Return aggregated results with comparison data                     │
│                                                                              │
│     def _apply_constraints(self, base_config, variant) -> dict:              │
